実装ポイント

  • プリセットは直接今日の記録へ出さず、マイ種目化してから選択対象にする。
  • 追加済みプリセットは `Exercise.sourcePresetId` で判定し、再追加不可にする。
  • マイ種目の削除は物理削除ではなく `isActive: false`。
  • 最近使った種目は `WorkoutExercise.createdAt` から派生算出する。

2. 画面操作設計

操作エラー条件表示メッセージタイミングと動作

部位を保存部位が選択されていない。部位を選択してください保存押下時に部位選択欄直下へ表示し、保存しない。
種目タイプを保存種目タイプが選択されていない。種目タイプを選択してください保存押下時に種目タイプ選択欄直下へ表示し、保存しない。
プリセットから追加追加対象が1件も選択されていない。追加する種目を1つ以上選択してください追加押下時に選択一覧の下へ表示し、シートを閉じない。
プリセットから追加選択後に別処理で同じ `sourcePresetId` が追加済みになった。選択した種目はすでにマイ種目へ追加されています登録直前に再検証する。重複分を登録せず、該当行を選択解除して追加済み表示へ切り替える。

複数項目にエラーがある場合はすべて表示し、画面内で最初のエラー項目へフォーカスする。入力または選択が正常になった項目のメッセージは即時解除する。

export async function addFromPresets(presetIds: UUID[]) {
  return db.transaction('rw', db.exercisePresets, db.exercises, async () => {
    const existing = await db.exercises
      .where('sourcePresetId')
      .anyOf(presetIds)
      .toArray();
    const existingIds = new Set(existing.map((exercise) => exercise.sourcePresetId));
    const presets = await db.exercisePresets.where('id').anyOf(presetIds).toArray();

    const nextSortOrder = await getNextExerciseSortOrder();
    const exercises = presets
      .filter((preset) => !existingIds.has(preset.id))
      .map((preset, index) => createExerciseFromPreset(preset, nextSortOrder + index * 10));

    await db.exercises.bulkAdd(exercises);
    return exercises;
  });
}
