Implementation Plan

筋トレ日記アプリ MVP 実装計画

詳細設計をもとに、React + TypeScript + Vite + Dexie で端末内保存の PWA を実装する。 今後の保守とレビューをしやすくするため、実装計画は画面ごとの HTML ファイルに分割する。

  • React + TypeScript
  • Vite PWA
  • IndexedDB / Dexie
  • Recharts
  • Client-only MVP

1. 実装方針

MVP はサーバ側処理を持たないクライアント完結の PWA として実装する。 記録データ、マイ種目、メニュー、設定は IndexedDB に保存し、グラフ集計はブラウザ内で行う。

最優先価値

今日の筋トレと体重を迷わず入力し、再起動後も確実に残ること。

設計原則

画面は Repository を経由してデータへアクセスし、将来の API 化や同期に備える。

完成条件

スマホ幅で主要操作が破綻せず、オフライン状態でも保存と閲覧が成立すること。

2. 技術スタック

領域 採用 実装上の注意
UI React + TypeScript 画面状態は feature 単位に分割し、フォーム入力は保存状態と分離する。
ビルド Vite クライアント完結の PWA として小さく始める。将来サーバ機能が必要になったら Next.js 移行を検討する。
保存 Dexie + IndexedDB 複数ストア更新は transaction に閉じ込める。
グラフ Recharts 集計済みの点配列を渡し、表示層で集計しない。
PWA vite-plugin-pwa アプリシェルを precache し、IndexedDB データは Service Worker で扱わない。

3. MVP範囲

MVPに含める

  • 今日の記録、日付移動、体重入力。
  • マイ種目作成、編集、非表示。
  • プリセットからマイ種目への複数追加。
  • メニューテンプレート作成、編集、削除。
  • メニューから今日の記録への一括追加。
  • 重量推移、体重推移グラフ。
  • 日別履歴、設定、データ初期化。

MVP後に回す

  • ログイン、クラウド同期、複数端末対応。
  • CSV エクスポート、インポート。
  • 推定1RM、総ボリューム、トレーニング分析。
  • 予定セット、目標重量、通知。

4. 実装フェーズ

1

土台構築

Vite、React Router、共通レイアウト、CSS tokens、PWA の初期設定を作る。

2

データ基盤

モデル、Dexie スキーマ、初期プリセット、Repository を実装する。

3

画面実装

今日の記録、種目、メニュー、グラフ、履歴、設定を画面別に実装する。

4

検証とPWA化

単体、Repository、E2E、スマホ表示、オフライン動作を確認する。

レビュー指摘Markdown生成

コメントは保存しない。入力内容からAIに渡すレビュー反映用のMarkdownを生成する。
