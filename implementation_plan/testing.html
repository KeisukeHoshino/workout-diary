2. E2E シナリオ

  1. プリセットからベンチプレスをマイ種目へ追加し、今日の記録へ追加する。
  2. 重量と回数を3セット入力し、再読み込み後も残っていることを確認する。
  3. 同じ種目をもう一度追加し、カードではなくセットが増えることを確認する。
  4. メニューを作成し、今日の記録へまとめて追加する。
  5. 重量推移グラフで同日最大重量だけが表示されることを確認する。
  6. 体重を入力し、体重推移グラフと日別履歴へ反映されることを確認する。

3. リスクと対策

リスク影響対策
IndexedDB 操作の複雑化保存漏れ、部分更新、削除漏れが起きる。Repository と transaction を徹底し、Repository 単体テストを厚めにする。
自動保存の競合古い保存結果が新しい入力を上書きする。保存リクエストに revision を持たせ、最新 revision だけ状態反映する。
スマホ画面の情報過多入力が重く感じる。主要操作は今日の記録に集約し、詳細編集はシート化する。
将来同期への移行ローカル専用設計が足かせになる。Repository 経由、UUID、ISO string、エンティティ分離を守る。
