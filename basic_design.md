# 筋トレ日記アプリ 基本設計書

## 1. 概要

本アプリは、日々の筋トレ内容と体重をスマホで素早く記録し、種目ごとの最大重量推移と体重推移を確認するための Web/PWA アプリケーションである。

MVP では、今日の記録、日付ごとの筋トレ記録、体重記録、マイ種目管理、プリセット追加、メニューテンプレート、重量推移・体重推移グラフ、端末内保存を対象とする。

対象ユーザーは、筋トレを継続したい個人、重量の伸びを確認したい人、ルーティン化したメニューを持つ人、体重も併せて管理したい人とする。

主要価値は、以下の3点である。

- 今日の記録を迷わず残せること。
- よく使う種目やメニューを事前に準備できること。
- 記録した結果がグラフで返ってくること。

## 2. システム構成

MVP は Web/PWA として構成する。スマホでの片手操作を主対象とし、デスクトップ表示は補助的に扱う。

保存方式は IndexedDB を採用する。記録、種目、メニュー、設定を端末内に保存し、基本的な利用はオフラインでも成立する前提とする。

ログイン、クラウド同期、複数端末対応、CSV エクスポートは MVP の対象外とする。ただし、将来のデータ連携に備え、保存データはエンティティごとに分離し、派生値は保存せず必要時に集計する。

## 3. 画面構成

MVP の主要画面は以下とする。

| 画面 | 役割 | 主な操作 |
| --- | --- | --- |
| 今日の記録 | 当日または選択日の体重・筋トレ記録を入力する。 | 体重入力、種目追加、メニュー追加、セット追加、重量・回数編集。 |
| 種目選択シート | マイ種目から今日の記録に追加する種目を選ぶ。 | 検索、部位絞り込み、最近使った種目選択、新規作成、プリセット導線。 |
| プリセット選択 | アプリ標準のプリセット種目からマイ種目を作る。 | 検索、部位絞り込み、複数選択、マイ種目へ追加。 |
| メニュー管理 | 1回分の筋トレメニューを作成・編集する。 | メニュー作成、種目追加、並び替え、編集、削除。 |
| メニュー選択シート | 今日の記録にメニュー内の種目をまとめて追加する。 | メニュー選択、メニュー作成導線。 |
| グラフ | 最大重量推移と体重推移を確認する。 | タブ切替、種目選択、期間変更、データ点詳細表示。 |
| 日別履歴 | 過去の記録を確認・編集する。 | 日付選択、記録確認、編集、削除。 |
| 設定 | アプリ全体の設定とデータ管理を行う。 | 単位確認、データ管理、将来のエクスポート導線。 |

## 4. 機能設計

### 4.1 今日の記録

- 初期表示は当日の日付とする。
- 前日・翌日へ移動できる。
- 日付をタップしてカレンダーから選択できる。
- 日付ごとに体重を入力・編集できる。
- 日付ごとに複数の種目カードを追加できる。
- 各種目カードには複数セットを追加できる。
- 各セットは重量 kg と回数を持つ。
- 入力内容は自動保存する。

### 4.2 種目管理

- ユーザーはマイ種目を自由に追加・編集できる。
- 種目には部位カテゴリを設定する。
- 種目には種別を任意で設定できる。
- 使わない種目は削除ではなく非表示にできる。
- マイ種目は今日の記録の種目選択シートに表示される。

### 4.3 種目プリセット

- アプリはプリセット種目一覧を持つ。
- ユーザーはプリセットから複数種目を選択し、マイ種目へ追加できる。
- 追加済みプリセットは再追加できない。
- プリセットは部位カテゴリと種目名で絞り込める。
- プリセットから追加した種目も通常のマイ種目として編集・非表示にできる。

### 4.4 メニューテンプレート

- ユーザーは1回分の筋トレメニューをテンプレートとして作成できる。
- メニューには複数のマイ種目を登録できる。
- メニュー内の種目は並び替えできる。
- 今日の記録画面からメニューを選択できる。
- メニューを選ぶと、含まれる種目が今日の記録にまとめて追加される。
- 追加された各種目には空の1セット目を作成する。
- メニューは編集・削除できる。

### 4.5 グラフ

- 重量推移と体重推移をタブで切り替える。
- 重量推移ではマイ種目を選択できる。
- 重量推移は、同一日・同一種目の全セットの中で一番重い重量を1点として表示する。
- 体重推移は、日付ごとの体重を1点として表示する。
- 期間は1か月、3か月、6か月、全期間から選べる。
- 初期期間は3か月とする。
- グラフ下にサマリーを表示する。
- データ点タップで詳細を表示する。

## 5. データモデル設計

### 5.1 共通方針

- ID は文字列 UUID とする。
- 日付はローカル日付の `YYYY-MM-DD` とする。
- 作成日時・更新日時は ISO 8601 文字列とする。
- 重量・体重は kg の number とする。
- 体重は小数1桁を許容する。
- 重量は小数1桁または 2.5kg 刻みを許容する。
- 未入力の空セットは `weightKg: null`、`reps: null` で保持する。

### 5.2 エンティティ

| 属性 | 型 | 説明 |
| --- | --- | --- |
| id | string | 固定の設定 ID。 |
| weightUnit | string | 重量・体重の単位。MVP では `kg`。 |
| defaultGraphRange | string | グラフ初期期間。MVP では `3m`。 |
| isSetupCompleted | boolean | 初回セットアップ完了状態。 |
| createdAt | string | 作成日時。 |
| updatedAt | string | 更新日時。 |

#### ExercisePreset

| 属性 | 型 | 説明 |
| --- | --- | --- |
| id | string | プリセット ID。 |
| name | string | 種目名。 |
| bodyPart | string | 部位カテゴリ。 |
| equipmentType | string \| null | 種別。 |
| sortOrder | number | 表示順。 |

### 5.3 リレーション

- `ExercisePreset` から選択された種目は `Exercise` として保存される。
- `Exercise` は `WorkoutExercise` と `MenuTemplateExercise` から参照される。
- `WorkoutDay` は複数の `WorkoutExercise` を持つ。
- `WorkoutExercise` は複数の `WorkoutSet` を持つ。
- `BodyWeightLog` は `WorkoutDay` に依存せず、日付単位で独立して保存する。
- `MenuTemplate` は複数の `MenuTemplateExercise` を持つ。
- メニューを今日の記録に追加すると、`MenuTemplateExercise` から `WorkoutExercise` と空の `WorkoutSet` を生成する。

## 6. データ保存設計

IndexedDB には、エンティティごとに独立したオブジェクトストアを用意する。

| オブジェクトストア | 主なキー | 主なインデックス |
| --- | --- | --- |
| userSettings | id | なし |
| exercisePresets | id | bodyPart, name |
| exercises | id | bodyPart, isActive, sourcePresetId |
| workoutDays | id | date |
| workoutExercises | id | workoutDayId, exerciseId |
| workoutSets | id | workoutExerciseId |
| bodyWeightLogs | id | date |
| menuTemplates | id | sortOrder |
| menuTemplateExercises | id | menuTemplateId, exerciseId |

入力変更時は自動保存する。保存ボタンは基本的に置かない。

削除操作には誤操作防止の確認を入れる。マイ種目は削除ではなく `isActive: false` による非表示を基本とする。記録やメニューは、ユーザーが明示的に削除した場合に削除する。

グラフ用の最大重量、期間内変化量、最新値などの派生データは保存せず、保存済みデータから表示時に集計する。

## 7. 業務ルール

| 項目 | ルール |
| --- | --- |
| 最大重量 | 同一日・同一種目の全セットの中で、一番重い重量をその日の最大重量とする。 |
| 同じ種目の追加 | 今日すでに同じ種目がある場合は、新しい種目カードを作らず既存カードに新しいセットを追加する。 |
| セット追加 | 既存セットがある場合は前セットの重量・回数をコピーする。セットがない場合は空の1セット目を作る。 |
| 体重記録 | トレーニング記録とは独立して日付ごとに保存する。トレーニングしない日も記録できる。 |
| プリセット追加 | プリセットは直接今日の記録へ出さず、マイ種目に追加された後に種目選択シートへ表示する。 |
| メニュー追加 | メニューを今日に追加した場合、含まれる種目カードと空の1セット目を生成する。 |
| グラフ初期表示 | 重量推移・体重推移ともに初期期間は直近3か月とする。 |
| 単位 | 重量と体重は kg を基本とする。 |

## 8. 非機能設計

### 8.1 ユーザビリティ

- スマホ片手操作を前提とする。
- 日々の入力は画面遷移を少なくする。
- 保存ボタンを押し忘れないよう自動保存を基本にする。
- 最近使った種目とメニューを優先表示する。

### 8.2 パフォーマンス

- 今日の記録画面は即時に表示されること。
- 種目検索とカテゴリ絞り込みは入力に追従すること。
- グラフ集計は通常利用の記録量でストレスなく表示されること。

### 8.3 データ保護

- MVP では端末内保存とする。
- 削除操作には誤操作を防ぐ確認を入れる。
- 将来のエクスポートやクラウド同期に備え、データ構造を分離しておく。

### 8.4 保守性

- プリセット、マイ種目、記録、メニューを別エンティティとして扱う。
- グラフ用集計は保存データから導出する。
- MVP 後の推定1RMや総ボリュームを追加しやすい構造にする。

## 9. テスト・受け入れ基準

| 番号 | 基準 | 確認観点 |
| --- | --- | --- |
| A-01 | 今日の記録画面で体重を入力できる。 | 入力後、同じ日付を再表示しても体重が残っている。 |
| A-02 | 今日の記録画面で種目を追加できる。 | マイ種目から選択した種目カードが日付画面に表示される。 |
| A-03 | セット単位で重量・回数を記録できる。 | 複数セットが保存され、編集後も保持される。 |
| A-04 | プリセットからマイ種目を追加できる。 | 追加した種目が種目選択シートに表示される。 |
| A-05 | メニューテンプレートから今日の記録へ複数種目を追加できる。 | メニュー内の種目カードが並び順通りに追加され、各種目に空の1セット目がある。 |
| A-06 | 重量推移グラフで日別最大重量を確認できる。 | 同日の複数セットから最大重量だけがグラフに反映される。 |
| A-07 | 体重推移グラフを確認できる。 | 体重記録が日付順に折れ線グラフで表示される。 |
| A-08 | グラフの初期期間が直近3か月である。 | 重量推移・体重推移ともに初回表示で3か月が選択されている。 |
| A-09 | 記録は端末内に保存される。 | アプリを閉じて再起動しても保存済みデータが残っている。 |
| A-10 | スマホ幅で画面が破綻しない。 | 主要画面でテキストやボタンが重ならず、操作可能である。 |
